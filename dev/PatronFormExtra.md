今天倒腾安卓APP突发奇想到一种离线验证是否拥有权限的方法  
利用RSA 公私秘钥无法互相推算的特性 首先签发一对秘钥 公钥放进JAR里 私钥由开发者保存 我再整一个签发脚本  

秘钥字段:
- 数据版本号
- 秘钥Type(int)
- 秘钥Version(int)
- 是否可以熔断(boolean) 除非特殊用途 否则必须为True
- 是否触发熔断(boolean) 除非特殊用途 否则必须为True
- 公钥
数据:
- 数据版本号
- 玩家 UUID
- 用户ID (客户端不使用 用于确定秘钥的所有者)
- 签发时间戳
- 时限 (比如3天 15天 1个月 3个月 不推荐太高的值 需要在程序里做一个最高值判断)
- 权限等级 (可以给不同等级用不同秘钥组)
- 额外数据 (用于实现独有形态)

之后把数据用私钥加密 生成.key文件 之后把.key放进config里就能实现离线鉴权  
之后由程序在config存储子公钥字段 用于实现熔断机制

额外安全加固方案
- 将数据Hash后根据hash选用秘钥组(可以提前备16个或256个秘钥) 防止私钥泄漏造成全面泄漏
- 私钥分为主私钥 和子私钥 Github或其他自动更新秘钥的用子私钥 利用签名机制(用主私钥给子公钥加密 子公钥加进.key文件里)
- 子密钥熔断机制 存储密钥字段里增加秘钥Type 秘钥Version 以及2个boolean(是否触发熔断 可以被熔断 默认都为True) 当服务器或客户端加载进更新的秘钥 自动废掉旧秘钥

秘钥存储
- 主公钥放JAR里 用于判断.key的秘钥是否可用
- 子公钥放进.key文件里 用于判断数据是否可用
- 主私钥由开发者保存 不可放进自动构建等地方 泄漏就损失大了 没法OTA熔断
- 子私钥由开发者保存 可以放进自动更新秘钥等地方 由开发者上传子私钥和已经打包完的秘钥字段 泄漏可以使用熔断机制来减少损失

如果需要实现全离线操作 需要将赞助者形态直接嵌进Mod里 这样就能实现全离线使用赞助者形态

文件结构(以二进制方式存储):
- 数据版本号 1x4(int)
- (密钥部分)
- (密钥部分长度 由于长度固定 不用写在文件里)
- 密钥类型 1x4(int)
- 密钥版本 1x4(int)
- 使用熔断机制 1x1(boolean)
- 公钥 4096位RSA公钥 1x512(byte[])
- 公钥数据签名 4096位RSA签名 1x512(byte[])
- (数据部分)
- 数据部分长度 1x4(uint) 最大为16(块)*512 bytes
- 玩家UUID 1x16(byte[])
- 签发时间戳 1x8(ulong)
- 时限 1x8(ulong)
- 权限等级 1x4(int)
- 额外数据数量 1x4(uint)
- (额外数据部分) 重复n次
- 额外数据key长度 1x4(uint)
- 额外数据key 1xN(byte[])
- 额外数据value长度 1x4(uint)
- 额外数据value 1xN(byte[])
- (总签名)
- 数据部分的bytes做签名 4096位RSA签名 1x512(byte[])

验证流程
- 客户端验证当前存储密钥是否有效 如果无效则删除已存储密钥 或其他提醒逻辑 并触发更新 如果没有已存储的密钥 则直接触发更新
- 更新逻辑: 根据玩家UUID下载对应密钥到config中 并验证是否有效 如果在服务器里则自动发送下载到的秘钥
- 进入服务器时向服务器发送客户端存储的密钥文件
- 服务器验证是否有效(服务器和客户端一样 都在config里存了最新密钥段 可以验证熔断) 如果密钥已被熔断则向客户端发送最新密钥段 如果新密钥触发了熔断 则向全部玩家发送最新密钥段 并提醒所有玩家30min后/服务器重启时当前密钥失效
- 客户端如果收到服务器的密钥段 验证是否有效 并触发本地熔断并触发更新
- 还得给30分钟的计时加一个提醒玩家更新 (进入时+每5min)

部署流程
- 我写一套python签发子密钥脚本 需要主私钥(我生成一份 再用微信或其他手段发送给onixary 双备份 防丢失 丢了旧版本就没法用赞助者功能了) 用来生成子私钥和公钥数据段
- 再写一套用子密钥(子私钥和公钥数据段)修改签发时间戳并签名(数据部分)的脚本 由CICD服务器定时触发更新密钥文件 或者由我或onixary手动签发

  - 密钥版本 1x4(int)

  - 公钥数据签名 4096位RSA签名 1x512(byte[])

  - 数据块:

      - 数据类型 1x4(int)
      - 数据版本 1x4(int)
      - 数据长度 1x4(int) 最大为8kb 后续可扩容
    - 实际数据 1xN(byte[]) N = `数据长度`


- 签名部分
  - 数据部分的bytes做签名 4096位RSA签名 1x512(byte[])


- 密钥段文件(可以随便改 反正仅影响签发)
  - 公钥段长度 1x4 (int)
  - 公钥段 1xN (byte[]) N = `公钥段长度` 结构 = `密钥部分`
  - 私钥长度 1x4 (int)
  - 私钥 1xN (byte[]) N = `私钥长度` DER编码的RSA私钥


- 赞助授权数据(Type = 数据块) Type = 1 Version = 0
  - 玩家UUID 1x16(byte[])
  - 权限等级 1x2(short)
  - 签发时间戳 1x8(long)
  - 时限 1x8(long)
  - 额外数据数量 1x2(short)
  - (额外数据部分) 重复`额外数据数量`次
    - 额外数据key长度 1x4(int)
    - 额外数据key 1xN(byte[]) N = `额外数据key长度`
    - 额外数据value长度 1x4(int)
    - 额外数据value 1xN(byte[]) N = `额外数据value长度`

.auth = 密钥部分+数据部分+签名部分 总大小不得超过1M
