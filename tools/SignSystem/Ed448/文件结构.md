当前使用的方案 需要java15+

- 密钥部分
  - 密钥部分长度 1x4(int)
  - 密钥类型 1x4(int)
  - 密钥版本 1x4(int)
  - 使用熔断机制 1x1(boolean)
  - 适用数据类型数量 1x4(int)
  - 适用数据类型 Nx4(int) N = `适用数据类型数量`
  - 公钥 Ed448公钥 1x57(byte[])
  - 公钥数据签名 Ed448签名 1x114(byte[])


- 数据部分
  - 数据段长度 1x4(int)
  - 数据个数 1x2(short)
  - 数据块:
    - Header:
      - 数据类型 1x4(int)
      - 数据版本 1x4(int)
      - 数据长度 1x4(int) 最大为8kb 后续可扩容
    - 实际数据 1xN(byte[]) N = `数据长度`


- 签名部分
  - 数据部分的bytes做签名 Ed448签名 1x114(byte[])


- 密钥段文件(可以随便改 反正仅影响签发)
  - 公钥段长度 1x4 (int)
  - 公钥段 1xN (byte[]) N = `公钥段长度` 结构 = `密钥部分`
  - 私钥长度 1x4 (int)
  - 私钥 1xN (byte[]) N = `私钥长度` Ed448 Raw 私钥 长度为57


- 赞助授权数据(Type = 数据块) Type = 1 Version = 0
  - 玩家UUID 1x16(byte[])
  - 权限等级 1x2(short)
  - 签发时间戳 1x8(long)
  - 时限 1x8(long)
  - 额外数据数量 1x2(short)
  - (额外数据部分) 重复`额外数据数量`次
    - 额外数据key长度 1x4(int)
    - 额外数据key 1xN(byte[]) N = `额外数据key长度`
    - 额外数据value长度 1x4(int)
    - 额外数据value 1xN(byte[]) N = `额外数据value长度`

.auth = MagicNumber+版本(固定为0(int))+密钥部分+数据部分+签名部分 总大小不得超过1M   
.key = 密钥部分   
.sub_key = 密钥段文件
